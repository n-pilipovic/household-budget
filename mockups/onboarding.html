Design notes

  • Create is the default path. The first card is highlighted with a brand-color border + ring. Most new users are setting up fresh; joiners are arriving via a deep link or sent code anyway.
  • Code format: 3+4, no ambiguous chars. FAM-7K2P uses an alphabet without 0/O/1/I/L. Short enough to read aloud, long enough to be unguessable. Case-insensitive on entry.
  • Share is two buttons, not one. "Copy code" for situations where you'll dictate or paste into chat; "Share link" opens the OS share sheet with a deep link that signs in + auto-joins on the partner's first launch.
  • Code-entry is a 7-cell input, not a single text field. Auto-advances per character, paste-detects a full code. Less error-prone than a free text field.
  • Helper card uses an info icon, not a banner. Soft tone, in surface-2. The user can read it if they're confused but it doesn't shout.
  • "Continue to budget" is the escape hatch. They can skip inviting now and do it later from settings — important when you're setting up for yourself first.
